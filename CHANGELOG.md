# 更新日志

本项目遵循 [语义化版本控制](https://semver.org/lang/zh-CN/)。

## [2.0.0] - 2026-07-13

### 新增
- 全栈 Web 应用架构（前后端分离）
- Spring Boot 2.7.18 后端 REST API
- Vue 3 + Vite + Element Plus 前端管理界面
- Docker Compose 一键部署（MySQL + Spring Boot + Nginx）
- 8 个 REST API 控制器（Auth, Child, Class, Course, Menu, Attendance, Transfer, Statistics）
- 前端路由管理（Vue Router）
- 前端 API 封装（Axios）
- 跨域配置（CorsConfig）
- 数据库健康检查
- Nginx 反向代理配置

### 优化
- 数据库连接通过环境变量配置
- 前端响应式设计
- 统一的错误处理机制
- 完善的日志记录

### 技术栈升级
- 后端：Spring Boot 2.7.18 / Java 8 / MySQL 8.0
- 前端：Vue 3 / Vite 4 / Element Plus 2
- 部署：Docker Compose

## [1.0.1] - 2026-07-08

### 修复
- 修复幼儿学籍模块"按班级查看"只能输入班级ID不能输入班级名称的问题
  - 现在支持输入班级编号（1-9）或班级名称（如"大一班"、"中班"等）
  - 新增 ClassDao.selectByName() 按名称模糊查询班级
  - 优化用户体验，增加输入提示
- 修复幼儿出生日期与班级不匹配问题：大班→2020年、中班→2021年、小班→2022年
- 修复所有幼儿都选了全部4门课的问题：改为随机分配2~4门课，产生差异化分布
- 修复每周食谱不合理问题：早餐=主食+水果，午餐=主食+荤菜+素菜+汤，晚餐=主食+素菜+汤
- 修复 ChildDao.insert() 未将生成ID回写到实体对象的问题
- 修复用户密码字段 VARCHAR(50) 不足以存储 SHA-256 哈希(64字符)的问题
- 修复 CourseServiceTest 中依赖"每人固定4门课"的过时假设
- 修复数据库密码与实际环境不一致的问题

### 新增
- 新增边界情况与CRUD多样性测试（EdgeCaseTest，19个测试用例）
  - 同名幼儿添加、搜索、删除互不影响
  - 批量添加不同年级幼儿
  - 修改幼儿信息验证
  - 出生日期与班级匹配验证
  - 重复选课/退未选课拒绝
  - 离园幼儿不能考勤/调班
  - 同一天重复考勤覆盖
  - 跨年级调班
  - 统计数据一致性
- 新增 BUG-FIX-LOG.md 问题追踪文档

## [1.0.0] - 2026-07-08

### 新增
- 幼儿学籍管理（增删改查、按班级查看、按姓名搜索）
- 课程管理（增删课程、选课退课、容量校验）
- 食谱管理（菜品库管理、每周排餐、复制上周食谱）
- 调班管理（跨年级调班、事务控制、调班记录）
- 考勤管理（逐个/批量打卡、历史查询、出勤率统计）
- 数据统计（班级人数、课程选课、年级分布、出勤率）
- 管理员和教师双角色权限控制
- 数据库自动初始化（建库建表 + 预置数据）
- 密码 SHA-256 哈希存储
- DAO 层统一异常处理（DataAccessException）
- 单元测试（8 个测试类）
- GitHub 协作规范（Issue/PR 模板、贡献指南、构建脚本）

### 技术栈
- Java 8+
- MySQL 8.x
- JDBC 直连（无 ORM 框架）
- 分层架构（View → Service → DAO → Entity）

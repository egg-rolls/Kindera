## 1. 文档目的

本文档明确幼儿园管理系统的**系统边界、功能范围、排除项和约束条件**，防止项目范围蔓延，确保团队聚焦核心交付。

## 3. 功能范围矩阵

### 3.1 ✅ 系统范围内（In Scope）

| 模块 | 包含功能 | 边界说明 |
|------|---------|---------|
| 登录认证 | 登录、菜单权限区分、修改密码 | 仅2种角色：管理员/教师 |
| 幼儿学籍 | 增删改查幼儿信息、按班级查看 | 支持在园/离园状态管理 |
| 班级管理 | 查看9个预置班级、查看班级幼儿 | 班级不可增删，仅查看和管理 |
| 课程管理 | 动态增删课程、选课/退课、容量校验 | 每人限4门，课程有容量上限 |
| 食谱管理 | 菜品库CRUD + 每周排餐 | 按周一~周五，早中晚三餐 |
| 调班管理 | 任意调班、调班记录、容量校验 | 支持跨年级，记录操作日志 |
| 考勤管理 | 每日打卡、历史查看、出勤率统计 | 教师仅管理本班 |
| 数据统计 | 班级人数/课程选课/出勤率报表 | Web界面图表展示 |

### 3.2 ❌ 系统范围外（Out of Scope）

| 排除项 | 原因 |
|--------|------|
| 家长端 / 家长查看功能 | 题目未要求，且超出系统范围 |
| 学费/财务管理 | 不在需求范围内 |
| 成绩/评分管理 | 幼儿园阶段无成绩概念 |
| 教师排课/课表管理 | 题目关注幼儿侧课程分配 |
| 家园联系/通知功能 | 超出系统边界 |
| 数据导入/导出（Excel等） | 非核心功能，时间有限 |
| 操作日志全记录 | 仅调班操作记录日志，其他不强制 |
| 数据备份/恢复 | 超出课程作业范围 |
| 多语言支持 | 仅中文 |
| 移动端适配 | 仅Web端 |

## 4. 用户边界

### 4.1 管理员操作边界

```
管理员可以：
  ✅ 登录/退出系统
  ✅ 增删改查幼儿信息
  ✅ 查看班级列表和班级幼儿
  ✅ 增删改查兴趣课程
  ✅ 管理菜品库
  ✅ 编辑每周食谱
  ✅ 执行调班操作
  ✅ 查看所有统计数据
  ✅ 修改自身密码

管理员不可以：
  ❌ 代替教师进行考勤打卡（教师专属）
  ❌ 删除预置班级
  ❌ 创建/删除其他管理员账号（系统预置）
```

### 4.2 教师操作边界

```
教师可以：
  ✅ 登录/退出系统
  ✅ 查看本班幼儿名单及详情
  ✅ 记录本班幼儿每日考勤
  ✅ 查看本班考勤历史
  ✅ 查看课程安排（自己负责的课程）
  ✅ 查看本周食谱
  ✅ 修改自身密码

教师不可以：
  ❌ 增删改幼儿信息
  ❌ 管理课程/食谱
  ❌ 执行调班操作
  ❌ 查看其他班级的数据
  ❌ 查看统计数据
```

## 5. 数据边界

### 5.1 数据量边界

| 数据类型 | 最大容量设计 | 说明 |
|---------|------------|------|
| 幼儿总数 | ≤ 150人 | 超出90人预设时仍可工作 |
| 单班人数 | ≤ 15人 | 默认10人，调班时校验上限 |
| 兴趣课程数 | ≤ 20门 | 管理员可动态添加 |
| 单课程人数 | ≤ 30人 | 默认15人，可配置 |
| 用户账号数 | ≤ 20个 | 管理员+教师 |
| 历史考勤 | 保留当前学期 | 跨学期数据不自动清理 |

### 5.2 数据生命周期

```
幼儿入园 → 录入学籍 → 分配班级 → 选择课程 → 日常考勤
         → [可能调班] → [可能退课/选课]
幼儿离园 → 状态标记为"离园" → 保留历史数据 → 不物理删除
```

## 6. 技术约束

| 约束项 | 具体要求 |
|--------|---------|
| 后端语言 | Java 8 |
| 后端框架 | Spring Boot 2.7.18 |
| 前端框架 | Vue 3 + Vite + Element Plus |
| 数据库 | MySQL 8.0 |
| 连接方式 | JDBC直连（无ORM框架） |
| 架构模式 | 前后端分离（REST API） |
| 部署方式 | Docker Compose |
| 代码注释 | 每个类必须有：类说明、作者、日期、版本 |

## 7. 接口边界

### 7.1 前后端接口规范

```
前端 (Vue 3) ──HTTP/JSON──▶ 后端 (Spring Boot) ──JDBC──▶ MySQL
    │                           │
    │ 只负责：                  │ 只负责：
    │ - 页面渲染                │ - REST API
    │ - 用户交互                │ - 业务校验
    │ - 数据展示                │ - 数据组装
    │ - 路由管理                │ - 事务管理
    │                           │
    │ 禁止：                   │ 禁止：
    │ - 直接操作数据库          │ - 页面渲染
    │ - 业务逻辑判断            │ - 用户交互
```

### 7.2 REST API 规范

| 路径 | 方法 | 说明 |
|------|------|------|
| `/api/auth/login` | POST | 用户登录 |
| `/api/children` | GET/POST/PUT/DELETE | 幼儿管理 |
| `/api/classes` | GET | 班级查询 |
| `/api/courses` | GET/POST/PUT/DELETE | 课程管理 |
| `/api/menus` | GET/POST/PUT/DELETE | 食谱管理 |
| `/api/attendance` | GET/POST | 考勤记录 |
| `/api/attendance/batch` | POST | 批量考勤 |
| `/api/transfers` | POST | 调班操作 |
| `/api/statistics/*` | GET | 数据统计 |

## 9. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| MySQL连接失败 | 系统无法使用 | 启动时检测连接，友好提示配置信息 |
| 课程容量冲突 | 并发选课超员 | 选课时实时查询当前人数并校验 |
| 删除幼儿后数据残留 | 数据不一致 | 删除时级联清理选课和考勤记录 |
| 前端API请求失败 | 页面无法加载 | 统一错误处理，友好提示 |
| Docker部署失败 | 服务无法启动 | 健康检查，日志记录 |
